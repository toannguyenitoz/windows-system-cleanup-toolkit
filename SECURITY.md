# Security Policy

## Supported usage

This toolkit is designed for lab environments, IT Support training and controlled troubleshooting.

## Reporting a vulnerability

If you find a security issue, please open a GitHub issue with:

- A short description
- Steps to reproduce
- Windows version
- PowerShell version
- Expected and actual behaviour

## Safe design principles

- Scan first.
- Confirm before cleanup.
- Do not delete user data automatically.
- Do not clean protected Windows folders.
- Keep reports for review.
